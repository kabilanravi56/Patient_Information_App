Patient Information App
REGISTER NO:732923ITR046
Problem Statement
In hospitals and clinics, maintaining patient information digitally helps reduce manual errors and improves communication between patients and doctors.
The problem is to design and develop an Android application named “Patient Information App” using Android Studio.
The application should:
•	Collect patient details
•	Display them on another screen
•	Provide communication options (Call, SMS, Email)
•	Show confirmation dialog before performing actions
Objectives
By developing this application, students will:
•	Understand Android app development using Java
•	Design interactive UI using XML layouts
•	Implement RadioButton, Spinner, DatePicker
•	Use Explicit Intent to move between activities
•	Use Implicit Intent for call, SMS, and email
•	Implement AlertDialog (Dialog Box)
•	Handle runtime permissions

Concepts Used
Concept-	Purpose
EditText	-Enter name, age, phone
RadioGroup & RadioButton-	Select gender
Spinner	-Select illness type
DatePickerDialog	-Select appointment date
Button-	Submit & communication actions
Explicit Intent-	Move to next activity
Implicit Intent-	Call, SMS, Email
AlertDialog	-Confirmation before action
AndroidManifest.xml	Permissions
